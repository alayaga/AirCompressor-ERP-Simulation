# 空压机虚仿平台 — 使用文档

## 1. 项目概述

**空压机虚仿平台（AirCompressor）** 是一个基于 Unity 引擎开发的制造业虚拟仿真培训系统。项目以空压机生产销售为业务场景，通过 3D 第一人称视角模拟企业完整的 ERP 业务流程——玩家扮演不同岗位角色（销售员、主管、仓管员、PMC、采购员、生产主管、质检员等），在虚拟工厂/办公室中与 NPC 交互、填写/审核/下推各类业务单据，完成从客户询单到产品交付的全链路业务操作。

- **引擎版本**: Unity（URP 渲染管线）
- **开发语言**: C#
- **解决方案文件**: `AirCompressor.sln`
- **自定义脚本**: ~80 个核心 C# 文件（不含 Unity 第三方插件和 Package 依赖）

## 3. 核心架构

### 3.1 流程引擎（Flow System）

整个项目的核心是一套基于协程的**业务流程引擎**，负责驱动所有业务操作的有序执行。

| 类名 | 文件路径 | 职责 |
|------|---------|------|
| `FlowBase` | `C#/Base/FlowBase.cs` | 所有流程的抽象基类。提供协程生命周期管理（Start/Stop）、玩家位置等待、单据面板打开等通用功能 |
| `FlowManager` | `C#/Base/FlowManager.cs` | 流程管理器（单例）。负责流程注册、启动、停止及队列调度，支持顺序/并行两种执行模式 |
| `FlowRunner` | `C#/Base/FlowRunner.cs` | 协程安全调度器。解决 StopFlow 后协程继续执行的问题 |
| `FlowStepBase` | `C#/Base/FlowStepBase.cs` | 流程步骤基类。定义 Execute/OnEnter/OnExit 生命周期 |
| `FlowContext` | `C#/Base/FlowContext.cs` | 流程上下文。携带客户需求数据及扩展数据，在步骤间传递信息 |
| `FlowStepTracker` | `C#/Base/FlowStepTracker.cs` | 静态步骤追踪器，提供 `CompleteStep()` 方法供全局调用 |
| `FlowTaskIntegration` | `C#/Base/FlowTaskIntegration.cs` | 流程-UI 集成管理器，桥接流程逻辑与任务引导面板 |
| `FlowStarter` | `C#/Flows/Steps/FlowStarter.cs` | 流程启动器，场景加载后自动启动流程 |

**流程生命周期**:
```
注册(RegisterFlow) → 启动(StartFlow) → 协程执行(FlowCoroutine)
                                              ↓
                              逐步骤: 等待玩家移动 → 等待交互(E键)
                              → 打开单据面板 → 等待填写/审核/签字
                              → 关闭面板 → MarkStepComplete → 下一步
                                              ↓
                                         结束(FinishFlow)
```

#### 单据配置（BillData ScriptableObject）

每张单据通过 `BillData` 资产进行配置，配置内容包括：
- **角色按钮权限**: 不同角色看到不同操作按钮（填写/审核/下推/查看/签字/发货/提交/保存）
- **预填数据**: 点击"填写"后自动填入的输入框内容和表格数据
- **预览数据**: 打开单据时不点"填写"也能看到的预览数据
- **弹窗/横幅文字**: 各类操作的提示语
- **签字区域配置**: 签字提示文字及固定内容

**匹配规则**: 角色名支持精确匹配和模糊匹配（如 Config"PMC"可匹配 Flow"PMC主管"，Config"工人"可匹配"1车间工人"/"2车间工人"）

**配置存放路径**: `Assets/Resources/BillConfigs/*.asset`

## 4. 关键子系统

### 4.1 交互系统

玩家通过**射线检测**（Raycast）看向场景中的 NPC/物体，屏幕中央显示 `"按E交互"` 提示，按下 E 键触发交互。

| 类名 | 职责 |
|------|------|
| `InteractionManager` | 单例管理器，监听 E 键，校验目标 NPC 是否匹配当前流程步骤，触发交互 |
| `PlayerInteractionDetector` | 挂在玩家上，通过射线检测前方的 `Interactables` 组件 |
| `Interactables` | 挂在场景物体/NPC 上，定义 NPC 名称、位置、操作类型（Fill/Approve/View 等） |

**交互校验流程**:
```
玩家按E → InteractionManager.TryInteract()
  → 检查前方是否有 Interactables
  → 获取当前流程的当前步骤目标NPC
  → 校验目标 NPC 是否匹配
  → 检查动画触发器（车间动画/质检动画）
  → 调用 target.OnInteract()
  → 流程完成当前步骤（FlowStepTracker.CompleteStep()）
```

### 4.2 玩家系统

| 类名 | 职责 |
|------|------|
| `SimpleFirstPersonController` | 第一人称角色控制器。支持移动（走路/跑步/蹲下）、跳跃、鼠标视角旋转、输入开关控制 |
| `CrosshairUI` | 屏幕准心，打开单据/对话时自动隐藏 |
| `InteractionPromptUI` | 交互提示文字 UI |

**操作控制**:
- **WASD**: 移动（Shift 跑步，C 蹲下）
- **鼠标**: 视角旋转
- **空格**: 跳跃 / 推进对话
- **E**: 交互（与 NPC/物体互动）
- **1/2 数字键**: 分支流程选择

## 6. 快速开始

### 6.1 环境要求

- Unity 编辑器（URP 支持）
- .NET / C# 开发环境
- 建议使用 Visual Studio 或 VS Code 作为代码编辑器

### 6.2 打开项目

1. 使用 Unity Hub 打开 `D:\Coding\AirCompressor` 文件夹
2. 等待 Unity 加载资源和编译脚本（6604 个 C# 文件，首次编译可能需要数分钟）
3. 在 `Assets/Scenes/` 中选择主场景打开

### 6.3 运行流程

#### 方式一：自动启动（默认）

场景加载后，`FlowStarter` 组件会在 2 秒延迟后自动启动流程：

1. 流程类型由 `DemandManager.currentWorkflowType` 决定（Standard/Custom）
2. 也可以通过 `FlowStarter` 的 `enableTestMode` 开启测试模式，手动选择要测试的流程

#### 方式二：手动测试

在 `FlowStarter` 组件上：
1. 勾选 `Enable Test Mode`
2. 从 `Test Flow Type` 下拉菜单选择要测试的流程
3. 运行场景

**可选的测试流程**:
- `CustomSales` — 定制销售
- `CustomProduction` — 定制生产
- `CustomPurchase` — 定制采购
- `StandardSales` — 标准销售
- `StandardSalesBranch` — 标准销售分支
- `StandardProduction` — 标准生产
- `StandardPurchase` — 标准采购
- `StandardDelivery` — 标准交付

### 6.4 玩法流程

1. **查看任务**: 屏幕左侧"任务引导面板"显示当前步骤说明和目标 NPC
2. **移动到目标**: 使用 WASD 移动到目标位置
3. **与 NPC 交互**: 面向目标 NPC，按 E 键交互
4. **对话（可选）**: 如果该步骤配置了对话，按空格键逐句推进
5. **操作单据**: 在弹出的单据面板上填写/审核/签字/下推/提交
6. **完成步骤**: 完成单据操作后自动进入下一步
7. **重复**: 依次完成所有步骤，直到流程结束

## 7. 自定义与扩展

### 7.1 添加新流程

1. 在 `Assets/C#/Flows/Steps/` 创建新类，继承 `FlowBase`
2. 实现 `FlowCoroutine()` 方法，在其中定义步骤队列
3. 使用 `InitializeSteps()` 初始化步骤
4. 使用 `WaitForBillComplete()` 打开单据面板并等待用户操作
5. 在 `FlowStarter.GetTestFlowType()` 中注册对应的枚举值和类型映射

### 7.2 添加新单据

1. 在 `UIManager.UIType` 枚举中添加新类型
2. 在 Unity Editor 中创建 `BillData` 资产：`Assets → Create → AirCompressor → BillData`
3. 配置单据名称、预填数据、角色按钮权限、弹窗文字等
4. 放入 `Assets/Resources/BillConfigs/` 目录
5. 在 `UIManager` 的 `uiEntries` 列表中绑定对应的 Panel 预制体

### 7.3 添加新对话

1. 在 Unity Editor 中创建 `DialogueData` 资产
2. 填充对话行（说话人和文本）
3. 放入 `Assets/Resources/Dialoguedata/` 目录
4. 在流程步骤的 StepData 中设置 `dialogueConfig`

### 7.4 配置角色按钮权限

在 `BillData` 资产的 `Role Settings` 中：
1. 添加角色名称（如：销售员、销售主管、PMC、仓管员、工人）
2. 勾选该角色在该单据上可见的操作按钮
3. 支持精确匹配和模糊匹配（如 Config"工人"可匹配"1车间工人"）

### 7.5 场景配置要点

1. **FlowStarter**: 场景中必须有挂载 `FlowStarter` 的空物体
2. **FlowManager**: 场景中必须有挂载 `FlowManager` 的空物体
3. **UIManager**: 场景中必须有挂载 `UIManager` 的空物体，且配置好所有 Panel 引用
4. **InteractionManager**: 场景中必须有挂载 `InteractionManager` 的空物体，绑定 `PlayerInteractionDetector`
5. **DemandManager**: 场景中必须有挂载 `DemandManager` 的空物体
6. **FlowTaskIntegration**: 场景中必须有挂载 `FlowTaskIntegration` 的空物体
7. **Interactables**: 每个 NPC 物体上需要挂载 `Interactables` 组件，配置 NPC 名称和操作类型
8. **玩家**: 场景中需要有 Tag 为 "Player" 的物体，挂载 `SimpleFirstPersonController` 和 `PlayerInteractionDetector`

## 9. 常见问题

### Q: 启动后看不到任何 UI 面板？
**A**: 检查场景中的 `UIManager` 是否已配置 `UI Entries` 列表，确保每个 Panel 预制体已拖入。

### Q: 按 E 无法交互？
**A**: 确认：
1. 面前物体挂载了 `Interactables` 组件
2. 当前流程步骤的目标 NPC 与该物体的 `npcName` 匹配
3. `InteractionManager` 中的 `PlayerInteractionDetector` 已正确绑定

### Q: 单据打开后一片空白？
**A**: 确认：
1. `Resources/BillConfigs/` 下有对应的 `BillData` 资产
2. 单据 Panel 预制体根节点挂载了 `BillView` 组件

### Q: 流程卡住不前进？
**A**: 查看 Console 日志输出（带 `[FlowBase]`、`[CustomSalesFlow]` 等前缀），通常会打印详细的错误信息指出缺少什么配置。
