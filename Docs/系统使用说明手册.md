## 1. 系统概述

### 1.1 软件简介

本系统是一款基于Unity引擎开发的**空压机企业ERP仿真教学软件**，模拟完整的企业运营流程，包括销售、生产、采购、仓储等核心业务环节。用户可以通过第一人称视角在虚拟工厂中体验各类业务单据的填写、审核和流转过程。

### 1.2 主要功能

| 功能模块 | 说明 |
|---------|------|
| 销售管理 | 报价单、销售合同、销售订单的完整流程 |
| 生产管理 | 生产工单、用料清单、工序计划与执行 |
| 采购管理 | 采购申请、采购订单、收料入库 |
| 仓储管理 | 库存管理、入库出库、领料发货 |
| 质量管理 | 来料检验、生产检验、成品检验 |

### 1.3 系统特点

- **沉浸式体验**：第一人称视角漫游工厂
- **完整流程**：覆盖企业运营全流程
- **任务引导**：实时任务面板指引操作
- **单据仿真**：真实的ERP单据填写体验

### 2.2 核心类说明

#### 2.2.1 FlowBase（流程基类）

所有业务流程的抽象基类，提供流程生命周期管理。

```csharp
public abstract class FlowBase
{
    // 流程事件
    public event Action OnFlowStart;   // 流程开始事件
    public event Action OnFlowEnd;     // 流程结束事件
    
    // 核心方法
    public virtual void StartFlow();   // 启动流程
    public virtual void StopFlow();    // 停止流程
    protected abstract IEnumerator FlowCoroutine();  // 流程协程（子类实现）
    
    // 辅助方法
    protected IEnumerator WaitForPlayerReachPosition(Vector3 targetPosition);
}
```

#### 2.2.2 FlowManager（流程管理器）

管理所有流程的注册、启动、停止和队列执行。

**主要功能：**
- 流程注册与注销
- 单流程启动/停止
- 多流程顺序执行
- 多流程并行执行

#### 2.2.3 单例管理器

| 管理器 | 职责 |
|--------|------|
| UIManager | 管理所有UI面板的显示/隐藏 |
| ObjectManager | 管理游戏中的关键物体（玩家、空压机等） |
| PositionManager | 管理场景中的关键位置点 |
| DemandManager | 管理客户需求数据 |
| InventoryManager | 管理零件库存 |

## 4. 业务流程详解

### 4.1 销售流程 (SalesFlow)

**流程步骤：**

```
┌─────────────────────────────────────────────────────────────┐
│                      销售流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐              │
│  │ 销售报价单 │ → │ 销售合同  │ → │ 销售订单  │              │
│  └──────────┘    └──────────┘    └──────────┘              │
│       │               │               │                     │
│       ↓               ↓               ↓                     │
│  填写→审核→下推   填写→审核→下推   填写→审核→下推           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**详细步骤：**

| 序号 | 步骤 | 操作人员 | 操作 |
|------|------|---------|------|
| 1 | 填写销售报价单 | 销售员 | 填写 |
| 2 | 审核销售报价单 | 销售主管 | 审核 |
| 3 | 下推销售报价单 | 销售员 | 下推到销售合同 |
| 4 | 填写销售合同 | 销售员 | 填写 |
| 5 | 审核销售合同 | 销售主管 | 审核 |
| 6 | 下推销售合同 | 销售员 | 下推到销售订单 |
| 7 | 填写销售订单 | 销售员 | 填写 |
| 8 | 审核销售订单 | 销售主管 | 审核 |
| 9 | 下推销售订单 | 销售员 | 完成销售流程 |

### 4.3 采购流程 (PurchaseFlow)

**流程步骤（共21步）：**

```
┌─────────────────────────────────────────────────────────────┐
│                      采购流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  库存检查 → 物料需求单 → 采购需求申请单 → 采购申请单         │
│                                                             │
│       ↓                                                     │
│                                                             │
│  采购订单 → 采购过程 → 收料通知单 → 质检 → 采购入库单        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**关键步骤说明：**

| 阶段 | 涉及单据 | 操作人员 |
|------|---------|---------|
| 库存检查 | 生产领料单 | 仓管员 |
| 需求申请 | 物料需求单 | 仓管员→仓库主管 |
| 采购申请 | 采购需求申请单、采购申请单 | 采购员→采购主管 |
| 采购执行 | 采购订单 | 采购员→采购主管 |
| 收料入库 | 收料通知单、采购入库单 | 采购员、仓管员 |
| 质量检验 | 来料检验 | 质检员 |

### 4.4 生产流程（第2部分）(ProductionFlow2)

**流程步骤（共26步）：**

```
┌─────────────────────────────────────────────────────────────┐
│                    生产执行流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  生产领料单 → 工序计划单 → 工序任务 → 生产准备              │
│                                                             │
│       ↓                                                     │
│                                                             │
│  前4道工序汇报 → 工序质检 → 第5道工序汇报 → 生产汇报单      │
│                                                             │
│       ↓                                                     │
│                                                             │
│  最终质检 → 成品入库                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**工序说明：**
- 共5道工序，由5个班组长负责
- 前4道工序完成后进行阶段质检
- 第5道工序（总装）完成后进行最终质检
- 质检合格后成品入库

## 5. 操作指南

### 5.1 启动系统

1. 启动Unity项目
2. 进入**开始界面**场景
3. 点击开始按钮进入主场景

### 5.2 基本操作

| 操作 | 按键/方式 |
|------|----------|
| 前进 | W |
| 后退 | S |
| 左移 | A |
| 右移 | D |
| 奔跑 | 按住Shift |
| 视角旋转 | 移动鼠标 |
| 打开菜单 | ESC |
| 交互/确认 | 鼠标左键 |

### 5.3 流程操作步骤

#### 5.3.1 查看任务引导

- 屏幕右上角显示**任务引导面板**
- 面板显示当前任务、目标NPC、位置和操作类型
- 点击折叠按钮可收起/展开面板

#### 5.3.2 前往目标位置

1. 查看任务引导面板中的**目标位置**
2. 使用WASD移动到指定位置
3. 到达后系统自动触发下一步

#### 5.3.3 填写表单

1. 到达指定NPC位置后，UI表单自动弹出
2. 点击**填写**按钮开始填写
3. 系统自动填充表单数据
4. 点击**提交**按钮完成填写

#### 5.3.4 审核表单

1. 前往审核人员位置
2. 查看表单内容
3. 点击**审核**按钮完成审核

#### 5.3.5 下推表单

1. 返回原操作人员位置
2. 点击**下推**按钮
3. 表单数据流转到下一环节

## 7. 常见问题

### 7.1 玩家无法移动

**可能原因：**
- 当前有UI面板打开
- 菜单处于打开状态

**解决方法：**
- 关闭当前UI面板
- 按ESC关闭菜单

### 7.2 任务面板不显示

**可能原因：**
- FlowTaskIntegration未初始化
- TaskGuidePanel引用丢失

**解决方法：**
- 确保场景中存在FlowTaskIntegration对象
- 检查TaskGuidePanel的引用设置

### 7.3 表单数据未显示

**可能原因：**
- InputBoxGenerator引用未设置
- SettingTableGenerator引用未设置

**解决方法：**
- 在Inspector中检查表单管理器的组件引用

### 7.4 流程无法继续

**可能原因：**
- 未到达指定位置
- 未点击正确的按钮

**解决方法：**
- 查看任务引导面板，前往指定位置
- 按照提示点击相应按钮
