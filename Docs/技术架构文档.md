## 2. 设计模式

### 2.1 单例模式 (Singleton)

系统中大量使用单例模式管理全局状态：

```csharp
public class UIManager : MonoBehaviour
{
    private static UIManager _instance;
    public static UIManager Instance
    {
        get
        {
            if (_instance == null) 
                Debug.LogError("UIManager 未初始化!");
            return _instance;
        }
    }
    
    private void Awake()
    {
        if (_instance == null)
            _instance = this;
        else
            Destroy(gameObject);
    }
}
```

**使用单例的类：**
- UIManager
- ObjectManager
- PositionManager
- DemandManager
- InventoryManager
- FlowManager
- FlowTaskIntegration
- TaskGuidePanel

### 2.2 模板方法模式 (Template Method)

`FlowBase` 使用模板方法模式定义流程框架：

```csharp
public abstract class FlowBase
{
    // 模板方法
    public virtual void StartFlow()
    {
        _isRunning = true;
        OnFlowStart?.Invoke();
        _currentCoroutine = _coroutineRunner.StartCoroutine(FlowCoroutine());
    }
    
    // 抽象方法 - 子类实现具体流程
    protected abstract IEnumerator FlowCoroutine();
    
    protected virtual void FinishFlow()
    {
        _isRunning = false;
        OnFlowEnd?.Invoke();
    }
}
```

### 2.3 观察者模式 (Observer)

使用C#事件实现流程状态通知：

```csharp
public abstract class FlowBase
{
    public event Action OnFlowStart;
    public event Action OnFlowEnd;
    
    public virtual void StartFlow()
    {
        OnFlowStart?.Invoke();  // 通知观察者
    }
}
```

### 2.4 策略模式 (Strategy)

不同流程类实现相同接口，可互换执行：

```csharp
// 基类定义接口
public abstract class FlowBase
{
    protected abstract IEnumerator FlowCoroutine();
}

// 具体策略实现
public class SalesFlow : FlowBase { ... }
public class ProductionFlow : FlowBase { ... }
public class PurchaseFlow : FlowBase { ... }
```


## 4. 协程使用规范

### 4.1 流程协程结构

```csharp
protected override IEnumerator FlowCoroutine()
{
    Debug.Log("流程开始");
    
    // 步骤1：等待玩家到达位置
    yield return WaitForPlayerReachPosition(targetPosition);
    
    // 步骤2：显示UI
    UIManager.Instance.ShowUI(UIManager.UIType.XXX);
    
    // 步骤3：等待用户操作
    yield return manager.WaitForFillButtonClick();
    
    // 步骤4：处理数据
    manager.SetQuoteFormData(data1, data2, tableData);
    
    // 步骤5：等待提交
    yield return manager.WaitForSubmitButtonClick();
    
    // 步骤6：隐藏UI并完成步骤
    UIManager.Instance.HideUI(UIManager.UIType.XXX);
    FlowStepTracker.CompleteStep();
    
    // 流程结束
    FinishFlow();
}
```

### 4.2 等待方法

```csharp
// 等待玩家到达位置
protected IEnumerator WaitForPlayerReachPosition(Vector3 targetPosition, float tolerance = 1.5f)
{
    while (Vector3.Distance(playerObject.transform.position, targetPosition) > tolerance)
    {
        yield return null;
    }
}

// 等待按钮点击
public IEnumerator WaitForFillButtonClick()
{
    ResetFillButtonClicked();
    while (!_isFillButtonClicked) 
        yield return null;
}
```

1. 创建新的流程类，继承 `FlowBase`：

```csharp
public class NewFlow : FlowBase
{
    protected override IEnumerator FlowCoroutine()
    {
        // 实现流程逻辑
        yield return null;
        FinishFlow();
    }
    
    public override void StartFlow()

        base.StartFlow();
        this.InitTaskUI();  // 初始化任务UI
```

## 7. 性能优化建议

### 7.1 对象池

对于频繁创建/销毁的UI元素，建议使用对象池：

```csharp
public class UIObjectPool
{
    private Queue<GameObject> pool = new Queue<GameObject>();
    
    public GameObject Get()
    {
        if (pool.Count > 0)
            return pool.Dequeue();
        return Instantiate(prefab);
    }
    
    public void Return(GameObject obj)
    {
        obj.SetActive(false);
        pool.Enqueue(obj);
    }
}
```

### 7.2 协程优化

避免在协程中频繁创建 `WaitForSeconds` 对象：

```csharp
// 不推荐
yield return new WaitForSeconds(0.5f);

// 推荐 - 缓存等待对象
private static WaitForSeconds wait05 = new WaitForSeconds(0.5f);
yield return wait05;
```

### 7.3 字典查找优化

使用 `TryGetValue` 替代 `ContainsKey` + 索引访问：

```csharp
// 不推荐
if (_uiDictionary.ContainsKey(uiType))
    return _uiDictionary[uiType];

// 推荐
if (_uiDictionary.TryGetValue(uiType, out GameObject uiObject))
    return uiObject;
```
