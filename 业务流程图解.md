# 空压机虚仿平台 — 业务流程图解

## 1. 总览：两大产品线

平台模拟空压机制造企业的 ERP 全链路，分为两条产品线，共享 5 条子流程。

## 2. 定制产品销售流程 (CustomSalesFlow)

**共 12 步骤，7 个业务阶段**  
核心单据: 销售订单、BOM单、销售报价单  
对话触发: 客户询单步骤自动播放对话

```
阶段1: 客户询单 (1步)
│
├── 步骤1: 客户询单         NPC:销售员      位置:销售办公室  操作:填写  单据:无  [对话]
│
阶段2: 销售订单处理 (2步)
│
├── 步骤2: 填写销售订单      NPC:销售员      位置:销售办公室  操作:填写  单据:SalesOrder
├── 步骤3: 审核销售订单      NPC:销售总监    位置:销售办公室  操作:审核  单据:SalesOrder
│         └─ 审核通过后自动下推 BOM 单
│
阶段3: BOM单处理 (1步)
│
├── 步骤4: 填写BOM单        NPC:技术员      位置:计划物控中心  操作:填写  单据:ProductionBOM
│
阶段4: 财务报价 (2步)
│
├── 步骤5: 查看BOM单        NPC:财务主管    位置:财务部      操作:查看  单据:ProductionBOM
├── 步骤6: 计算成本并确认    NPC:财务主管    位置:财务部      操作:填写  单据:SalesQuotation
│
阶段5: 报价确认 (3步)
│
├── 步骤7: 查看报价单        NPC:销售员      位置:销售办公室  操作:查看  单据:SalesQuotation
├── 步骤8: 报价给客户        NPC:销售员      位置:销售办公室  操作:填写  单据:SalesQuotation
├── 步骤9: 客户确认          NPC:销售员      位置:销售办公室  操作:查看  单据:无
│
阶段6: 提交PMC (1步)
│
├── 步骤10: 提交订单给PMC    NPC:销售员      位置:销售办公室  操作:填写  单据:SalesOrder
│
阶段7: 分支选择 (反复循环)
│
├── 步骤11: 选择后续流程      [系统步骤]
│         ├─ 按 [1] → CustomProductionFlow (定制生产流程, 57步)
│         └─ 按 [2] → CustomPurchaseFlow   (定制采购流程, 13步)
│         (可反复选择，两个分支都完成后自动结束)
│
└── 流程结束
```

## 3. 定制产品生产流程 (CustomProductionFlow)

**共 57 步骤，4 个生产车间 + PMC排产阶段**  
核心单据: 周计划、排产单、生产工单、派工单、领料单、退料入库单、工序汇报单  
车间顺序: 弯管(1)→焊接(2)→配电(3)→总装(4)，流水线依赖

│ 3. PMC填写每日排产单     PMC主管   计划物控中心  填写  Schedule    │
│ 4. 仓管员发料到备料区    仓管员A   备料区      交付  无          │
└────────────────────────────────────────────────────────────────┘

**领料/退料规则**: 仅弯管车间(1车间)执行领料、退料流程；焊接(2)、配电(3)、总装(4)车间跳过此步骤。

## 5. 标准产品主流程 (StandardSalesFlow)

**共 6 步骤，5 个可选分支，自由选择完成顺序**  
核心设计: 主流程只有分支选择步骤，实际业务逻辑在 5 条子流程中

```
                      ┌─────────────────────┐
                      │ 步骤1: 流程开始       │
                      │ [系统自动步骤, 1.5秒] │
                      └──────────┬──────────┘
                                 │
           ┌─────────────────────┼─────────────────────┐
           │                     │                     │
           ▼                     ▼                     ▼
   ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
   │ 步骤2: 选择分支 │   │ 步骤3: 选择分支 │   │ 步骤4: 选择分支 │
   │ (首次选择)     │   │ (第二次选择)    │   │ (第三次选择)    │
   └───────┬───────┘   └───────┬───────┘   └───────┬───────┘
           │                     │                     │
           └─────────────────────┼─────────────────────┘
                                 │
           ┌─────────────────────┼─────────────────────┐
           │                     │                     │
           ▼                     ▼                     ▼
   ┌───────────────┐   ┌───────────────┐   ┌───────────────┐
   │ 步骤5: 选择分支 │   │ 步骤6: 流程结束 │   │               │
   │ (第四次选择)    │   │ [系统自动步骤]  │   │               │
   └───────┬───────┘   └───────────────┘   └───────────────┘
           │
           │ 每个分支选择步骤均可按 [1][2][3][4][5] 选择:
           │
           ├── [1] → StandardSalesBranchFlow  (销售-PMC流程, 5步)
           ├── [2] → StandardDeliveryFlow     (销售-发货流程, 14步)
           ├── [3] → StandardProductionFlow   (标准生产流程, ~40步)
           ├── [4] → StandardPurchaseFlow     (标准采购流程, 12步)
           └── [5] → ProductionDeptPurchaseFlow (生产部采购, 10步)
```

**分支完成追踪**: 已选择并完成的分支在后续选择步骤中不再显示，5 个全完成后自动结束。

## 6. 标准销售-PMC流程 (StandardSalesBranchFlow)

**共 5 步骤**  
核心单据: 月度销售计划、周生产计划、两周采购计划  
角色: 销售总监 → PMC主管

```
步骤1: 填写月度销售计划    销售总监  → 销售办公室   → 填写  MonthlySalesPlan
  │
  ▼
步骤2: 查看销售计划         PMC主管  → 计划物控中心 → 查看  (无单据)
  │       (查看销售计划、订单、库存、在制、在途)
  ▼
步骤3: 回复交期             PMC主管  → 计划物控中心 → 填写  (无单据)
  │       (根据生产与采购周期向销售部回复预计交付时间)
  ▼
步骤4: 制作一周生产计划     PMC主管  → 计划物控中心 → 填写  WeeklyProductionPlan
  │
  ▼
步骤5: 制作两周采购计划     PMC主管  → 计划物控中心 → 填写  BiweeklyPurchasePlan
  │
  ▼
流程结束 → 返回主流程分支选择
```

## 7. 标准销售-发货流程 (StandardDeliveryFlow)

## 8. 标准产品生产流程 (StandardProductionFlow)

**共 ~40 步骤，复用定制生产流程结构**  
核心单据: 周生产计划、排产单、生产工单、派工单、领料单、退料入库单、工序汇报单  
与定制生产流程的差异: 标准生产由 PMC 制定周计划+排产单开始（无需查看销售订单），后续车间流程结构与定制生产相同

```
┌──────────────────────────────────────────────────┐
│                流程步骤统一执行模型                   │

│                                                  │

│  2. 判断步骤类型:                                   │
│     ├─ targetNPC == "系统" → 自动完成 (1.5~5秒)     │
│     ├─ targetNPC == "客户"/"供应商" → 自动完成 (5秒) │
│     └─ 其他 → 等待玩家按 E 交互                      │

│  3. 如果有 billType:                               │
│     └─ WaitForBillComplete()                      │
│        ├─ BillDataLoader.GetConfig()              │

│                                                  │
│  4. UpdateProgressToUI()                          │
│                                                  │
│  5. 循环至下一步骤                                   │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 13.2 操作类型与按钮映射

| ActionType | 单据按钮表现 | 典型场景 |
|-----------|-------------|---------|
| Fill | 显示填写+保存+提交+退出 | 创建/编辑单据 |
| Approve | 显示审核+下推+退出 | 主管审核并下推 |
| View | 显示退出（只读） | 查看他人单据 |
| Ship | 显示发货+退出 | 销售订单点发货 |
| Sign | 显示签字+退出 | 工人签字确认 |
| Pick | 无单据 | 领取物料 |
| Deliver | 无单据 | 实物交付/运输 |

### 13.3 自动下推规则

部分审核步骤在审核通过后自动触发"下推"操作，生成下游单据：

- 审核销售订单 → 自动下推 BOM 单
- 审核采购申请 → 自动下推（采购订单可见）
- 填写收料通知 → 自动下推（来料检验单可见）
- 填写来料检验 → 自动下推（采购入库单可见）
- 销售订单发货 → 自动下推 发货通知单
